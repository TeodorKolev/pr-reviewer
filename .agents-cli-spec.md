# Agent Spec — PR Guardian

## Overview

PR Guardian is a stateless multi-agent system built on ADK 2.0 that analyses
GitHub Pull Requests and produces structured approval recommendations for human
reviewers.

The system is read-only and advisory. It never approves, rejects, merges, or
modifies a PR. The final decision always belongs to the human reviewer.

### Agent architecture

- `pr_guardian` (coordinator): entry point, validates input, orchestrates pipeline
- `specialist_panel` (ParallelAgent): runs 4 specialists concurrently
  - `code_review_agent` → CodeAnalysisResult
  - `security_review_agent` → SecurityAnalysisResult
  - `tests_review_agent` → TestsAnalysisResult
  - `context_review_agent` → ContextAnalysisResult
- `synthesizer_agent` → PRRecommendation (final output)

## Example Use Cases

### Input
```
Analyse this PR: https://github.com/google/adk-python/pull/42
```

### Expected output
A structured PRRecommendation with:
- recommendation: "ready_for_review" | "needs_work" | "blocked"
- confidence: "high" | "medium" | "low"
- blocking_issues: list of critical issues (empty if ready)
- suggestions: list of non-blocking improvements
- strengths: what the PR does well
- summary: 3–5 sentence narrative
- disclaimer: advisory-only statement

## Tools Required

| Tool | Purpose | GitHub API | Auth |
|---|---|---|---|
| fetch_pr_metadata | PR header info | GET /repos/{owner}/{repo}/pulls/{number} | GitHub PAT |
| fetch_pr_diff | Full unified diff | GET /repos/{owner}/{repo}/pulls/{number} (diff Accept) | GitHub PAT |
| list_changed_files | File list with stats | GET /repos/{owner}/{repo}/pulls/{number}/files | GitHub PAT |
| fetch_file_content | Full file for context | GET /repos/{owner}/{repo}/contents/{path} | GitHub PAT |
| fetch_ci_status | Check runs / CI | GET /repos/{owner}/{repo}/commits/{sha}/check-runs | GitHub PAT |
| fetch_dependency_changes | Dep manifest diffs | derived from diff | GitHub PAT |
| fetch_pr_comments | Review discussion | GET /repos/{owner}/{repo}/pulls/{number}/comments | GitHub PAT |

All tools are GET-only. The `ReadOnlyEnforcerPlugin` blocks mutating calls at runtime.

## Constraints & Safety Rules

1. **Never call GitHub POST/PUT/PATCH/DELETE endpoints** — enforced by ReadOnlyEnforcerPlugin
2. **Never approve, reject, merge, or comment on a PR** — enforced by coordinator instruction
3. **Always include the advisory disclaimer** — built into PRRecommendation schema default
4. **Never fabricate findings** — instructions explicitly prohibit hallucinating issues
5. **Truncate large diffs** — max 200KB diff, 100KB per file to protect context window
6. **Draft PRs** — analyse normally but note draft status prominently

## Success Criteria

- Given a valid PR URL, produce a PRRecommendation within 60 seconds
- Correctly identify hardcoded secrets in diffs (critical security finding)
- Correctly flag missing test coverage when source files change without test files
- Produce `blocked` recommendation when CI checks are failing
- Produce `ready_for_review` recommendation for a clean, well-tested PR
- Never call mutating GitHub APIs (verified by ReadOnlyEnforcerPlugin)

## Deployment

- **Target**: Google Cloud Run
- **Session storage**: In-memory (stateless per request)
- **CI/CD**: GitHub Actions
- **Auth**: GITHUB_TOKEN in GCP Secret Manager
